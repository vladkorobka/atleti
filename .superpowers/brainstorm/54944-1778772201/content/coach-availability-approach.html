Як тренер керує доступністю?

Вкладка Calendar — яка модель управління слотами?

A

Розклад + ручне блокування

Тренер один раз налаштовує робочий графік (пн-нд, час початку/кінця, тривалість слоту). Потім у конкретні дні може заблокувати окремі слоти (обід, відпустка тощо). Клієнти бачать лише незаблоковані слоти.

Плюси

  • Один раз налаштував — завжди працює
  • Менше рутини

Мінуси

  • Потрібна окрема схема для блоків
B

Тільки явні слоти на конкретні дати

Тренер щодня/щотижня вручну "відкриває" потрібні слоти. Немає шаблону — лише конкретні дати і час. Клієнти бачать лише те, що тренер відкрив.

Плюси

  • Максимальний контроль
  • Простіша модель даних

Мінуси

  • Рутина — треба відкривати слоти щотижня
  • Більше ручної роботи
C

Шаблон + виключення + блоки

Тренер налаштовує повторюваний шаблон (робочі години на кожен день тижня). Може додавати виключення на конкретні дати (вихідний або додатковий день). Може блокувати окремі слоти всередині дня.

Плюси

  • Найгнучкіший варіант
  • Підходить для відпусток, свят

Мінуси

  • Найскладніша реалізація
